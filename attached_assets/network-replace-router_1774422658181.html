Network — Replace Router

Caution — Customer Downtime

Replacing a router will cause downtime for all customers connected through it. Schedule this during off-peak hours. Export the router config backup before proceeding.

Select Routers

Old Router (to replace)

New Router (replacement)

Replacement Checklist

1. Export Old Router Configuration

Run in MikroTik terminal on the old router:

/export file=backup-old-router
# Then download backup-old-router.rsc from Files

2. Note Customer IP Assignments & Queue Names

Verify all queue rules are synced in Queues before shutting down the old router.

3

3. Flash / Reset New Router

Reset the new router to factory defaults and apply a basic config:

/system reset-configuration no-defaults=yes
# After reboot — set IP:
/ip address add address=10.0.3.1/24 interface=ether1
/ip route add gateway=10.0.3.254
4

4. Import Config & Apply Scripts

Upload the backup .rsc and import it, then apply the hotspot or PPPoE self-install script from Self Install.

5

5. Update Router Record in System

After the new router is connected, update the router record in Routers with the new IP address and credentials, and verify it comes online.

6

6. Re-push Customer Queues

Go to Queues and click "Sync to Router" to repush all queue rules to the new router.

Execute Replacement

This will update the router IP in all customer queue assignments, re-sync RADIUS settings, and mark the old router as decommissioned.
